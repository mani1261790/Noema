VAEと拡散モデル

世界モデルでは、状態を圧縮して予測しつつ、必要なら高品質に観測を復元する必要があります。VAEと拡散はこの目的に対して異なる強みを持ちます。

背景と目的

VAEは潜在空間の構造化と高速な推論が得意で、拡散は高品質生成に強い傾向があります。

両者の役割分担を理解できると、計画用潜在と表示用観測を分離した設計が可能になります。

ここでは2次元データを1次元潜在に圧縮し、ノイズ除去復元を比較します。

最初に解きたい疑問

  1. ここでの VAE風 は、実際の VAE と何が違うのか。
  2. どうして PCA で encoder/decoder を代用しているのか。
  3. 潜在空間 にノイズを足してから戻すと、何が分かるのか。
  4. score と書かれているものは、何の勾配なのか。
  5. なぜ圧縮向きと高品質復元向きで、VAE と拡散の向き不向きが違うのか。

先に押さえる言葉

  • 潜在空間: 元データを圧縮して表した低次元の空間です。モデルが扱いやすい形にした内部表現です。
  • PCA: データのばらつきが大きい方向を使って圧縮する手法です。ここでは encoder/decoder の近似として使っています。
  • スコア: ある点をどちらに動かすと確率が高くなるかを示す量です。拡散モデルで使われる考え方です。
  • デノイズ: ノイズを取り除いて、元の形に近づけることです。復元や生成の途中で使います。

  • 誤差の小ささがそのまま生成品質ではない点。

コード 1: 入力データを作る

このセルでは 入力データを作る ための最小コードを動かします。 実行時は「z が 1 次元の潜在として作れているか。」を意識して、変数名と出力の対応を追ってください。

import numpy as np
np.random.seed(3)

n = 300
theta = np.random.rand(n) * 2 * np.pi
x = np.stack([np.cos(theta), np.sin(theta)], axis=1) + 0.08 * np.random.randn(n, 2)

コード 2: 入力データを作る

このセルでは 入力データを作る ための最小コードを動かします。 実行時は「VAE-like reconstruction MSE が出て、再構成誤差として読めるか。」を意識して、変数名と出力の対応を追ってください。

# VAE風: PCAを encoder/decoder の近似として利用
mu = x.mean(axis=0, keepdims=True)
xc = x - mu
u, s, vt = np.linalg.svd(xc, full_matrices=False)
pc1 = vt[0:1].T  # 2x1
z = xc @ pc1      # encoder
x_rec = z @ pc1.T + mu  # decoder
vae_mse = np.mean((x_rec - x) ** 2)

# Diffusion風: 潜在にノイズを加え、簡易スコア(ガウス事前)で反復デノイズ
z_noisy = z + 0.5 * np.random.randn(*z.shape)
z_den = z_noisy.copy()
prior_var = np.var(z_noisy) + 1e-8
for _ in range(30):
    score = -z_den / prior_var  # log N(0,var) の勾配
    z_den = z_den + 0.04 * score
x_den = z_den @ pc1.T + mu
diff_mse = np.mean((x_den - x) ** 2)

print('VAE-like reconstruction MSE       =', round(vae_mse, 6))
print('Diffusion-like latent denoise MSE =', round(diff_mse, 6))

ここでの VAE は PCA を使った近似、拡散は簡易スコア反復の近似です。圧縮主体なら VAE 系、高品質復元主体なら拡散系が有利になりやすいという設計感覚を掴むことを目的にしています。
