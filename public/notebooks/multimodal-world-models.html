マルチモーダルな世界モデル

実環境では、画像特徴・言語指示・行動履歴を同時に使って次状態を予測する必要があります。マルチモーダル融合は、この情報統合を行う中心技術です。

背景と目的

単一モダリティに依存すると、観測欠損やノイズで予測が壊れやすくなります。

融合モデルを作れると、どれかの情報が弱い場面でも頑健な予測が可能になります。

ここでは画像特徴・テキスト特徴・行動を結合した線形予測器を比較します。

最初に解きたい疑問

  1. 画像特徴、言語特徴、行動 を一緒に入れると何が良いのか。
  2. 単独のモダリティだけだと、なぜ予測が壊れやすいのか。
  3. fit_linear は何を学習していて、なぜ線形で比べているのか。
  4. MSE image-only と MSE multimodal の差は、何を意味するのか。
  5. どのモダリティがどの次状態成分に効いているのか。

先に押さえる言葉

  • マルチモーダル: 複数種類の情報源を同時に使うことです。画像、テキスト、行動などをまとめて扱います。
  • フュージョン: 異なる特徴をまとめて 1 つの予測に使うことです。情報を統合する役割があります。
  • 特徴量: モデルが入力として扱う数値表現です。元のデータをそのままではなく、扱いやすくしたものです。
  • 頑健性: 一部の情報が欠けても壊れにくい性質です。実環境での安定性に直結します。

実行前の見取り図

  1. MSE image-only、MSE text-only、MSE multimodal が並んで出るか。
  2. multimodal の誤差が単独入力より小さくなるか。
  3. y が 2 次元の次状態として構成されているか。

出力の読み方

  • MSE image-only と MSE multimodal の差の読み方。
  • どの入力が効いたかを数値でどう追うか。

つまずきやすい点

  • 各モダリティが y のどの成分に効いているのかをもう少し明示する説明。
  • これは線形予測器の比較であって、実際の深層融合とは違う注意書き。
  • 各モダリティが次状態のどこに効くかの明示。
  • これは深層融合ではなく線形比較である注意。

このノートの守備範囲

このノートでは次の点は入口だけ触れるか、別ノートに分けて扱います。

  • ここでは融合の入口だけを扱い、実際の深層マルチモーダル融合ではない点。

コード 1: 入力データを作る

このセルでは 入力データを作る ための最小コードを動かします。 実行時は「MSE image-only、MSE text-only、MSE multimodal が並んで出るか。」を意識して、変数名と出力の対応を追ってください。

import numpy as np
np.random.seed(9)

n = 240
img_feat = np.random.randn(n, 2)
text_feat = np.random.randn(n, 1)
action = np.random.randn(n, 1)

# 真の次状態（2次元）
y = (
    0.7 * img_feat
    + np.hstack([0.5 * text_feat, -0.3 * text_feat])
    + np.hstack([0.4 * action, 0.2 * action])
    + 0.05 * np.random.randn(n, 2)
)

コード 2: 更新式や補助関数を定義する

このセルでは 更新式や補助関数を定義する ための最小コードを動かします。 実行時は「multimodal の誤差が単独入力より小さくなるか。」を意識して、変数名と出力の対応を追ってください。

def fit_linear(X, Y):
    X1 = np.hstack([X, np.ones((len(X), 1))])
    W = np.linalg.pinv(X1) @ Y
    pred = X1 @ W
    mse = np.mean((pred - Y) ** 2)
    return mse

mse_img = fit_linear(img_feat, y)
mse_text = fit_linear(text_feat, y)
mse_multi = fit_linear(np.hstack([img_feat, text_feat, action]), y)

print('MSE image-only =', round(mse_img, 6))
print('MSE text-only  =', round(mse_text, 6))
print('MSE multimodal =', round(mse_multi, 6))

融合モデルは単独入力より誤差が下がることが多く、世界モデルの頑健性を高める基本戦略になります。
