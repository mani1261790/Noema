状態空間モデルとTransformer

長い系列を扱うとき、状態空間モデル（SSM）とTransformerは異なる計算原理で情報を集約します。ここでは両者の挙動差を小さい系列で確認します。

背景と目的

SSMは逐次更新でメモリ効率が高く、Transformerは自己注意で長距離依存を直接参照しやすい特徴があります。

どちらが有利かは系列長・必要な依存距離・計算予算で変わります。

同じ系列に対して2つの表現を作り、予測誤差を比較します。

最初に解きたい疑問

  1. SSM と Transformer は、情報を集める仕組みがどう違うのか。
  2. y_target は何を正解として作っているのか。
  3. x[5] と x[25] のイベントが、なぜ長距離依存の例になるのか。
  4. 自己注意 の重みは何を見て決まっているのか。
  5. SSM-like MSE と Transformer-like MSE のどちらをどう解釈すればいいのか。

先に押さえる言葉

  • 状態空間モデル: 逐次更新で系列を処理するモデルです。過去の情報を内部状態にたくわえていきます。
  • 自己注意: 各時点が過去のどこをどれだけ見るかを重み付けする仕組みです。遠い過去を直接参照できます。
  • 長距離依存: 遠い過去の情報が今の予測に効くことです。系列が長いほど重要になります。
  • 更新コスト: 1ステップ処理するために必要な計算の重さです。長い系列では設計上の制約になります。

実行前の見取り図

  1. SSM-like MSE と Transformer-like MSE が両方表示されるか。
  2. x の 5番目と 25番目のイベントが、結果に影響していそうか。
  3. y_target と各予測のズレを MSE で比較できるか。

出力の読み方

  • MSE SSM と MSE Transformer をそのまま優劣と読まない注意。
  • 長距離依存がどこで効いているかの見方。

つまずきやすい点

  • ここでの Transformer が簡略版であることの説明。
  • MSE の差が、そのまま本物のモデル優劣を意味しない注意書き。
  • このノートの Transformer が簡略代理である注意。
  • MSE 比較が本物の性能順位ではない説明。

このノートの守備範囲

このノートでは次の点は入口だけ触れるか、別ノートに分けて扱います。

  • ここでは小さな教材用比較であり、実装済み Transformer の代表性能ではない点。

コード 1: 入力データを作る

このセルでは 入力データを作る ための最小コードを動かします。 実行時は「SSM-like MSE と Transformer-like MSE が両方表示されるか。」を意識して、変数名と出力の対応を追ってください。

import numpy as np
np.random.seed(2)

T = 40
x = np.zeros(T)
x[5] = 1.0
x[25] = 0.8  # 長距離イベント

y_target = np.zeros(T)
for t in range(1, T):
    y_target[t] = 0.7 * y_target[t - 1] + x[t]

コード 2: 入力データを作る

このセルでは 入力データを作る ための最小コードを動かします。 実行時は「x の 5番目と 25番目のイベントが、結果に影響していそうか。」を意識して、変数名と出力の対応を追ってください。

# SSM: h_t = a h_{t-1} + b x_t, y_t = c h_t
a, b, c = 0.8, 1.0, 1.0
h = 0.0
y_ssm = np.zeros(T)
for t in range(T):
    h = a * h + b * x[t]
    y_ssm[t] = c * h

# Transformer風: 位置に依存した自己注意で過去を加重平均
idx = np.arange(T)
y_attn = np.zeros(T)
for t in range(T):
    past = x[: t + 1]
    dist = (t - idx[: t + 1]).astype(float)
    score = -0.15 * dist  # 近いほど重い単純な注意
    w = np.exp(score - score.max())
    w /= w.sum()
    y_attn[t] = np.dot(w, past)

mse_ssm = np.mean((y_ssm - y_target) ** 2)
mse_attn = np.mean((y_attn - y_target) ** 2)
print('SSM-like MSE        =', round(mse_ssm, 6))
print('Transformer-like MSE=', round(mse_attn, 6))

SSMは更新コストが低く長系列に有利です。ここでの Transformer 側は、学習された content-based attention ではなく距離に応じた positional smoothing を使う簡略代理です。そのため、この MSE 比較は『本物の Transformer が優れているか』を測る実験ではなく、系列集約の考え方の違いを見るための教材用比較です。
